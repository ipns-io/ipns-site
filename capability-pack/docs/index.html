launch: primary names only

For launch, IPNS supports primary names only. Subnames are intentionally out of scope until a hardened v2 design is implemented and audited.

Contract surface

register(string name, uint8 numYears)
renew(string name, uint8 numYears)
setCID(string name, string cid)
resolve(string name) returns (string)
getPrice(string name, uint8 numYears)

Subname operations (including resolveSub, setSubCID, and clearSubCID) are deferred to v2 and are unsupported for launch.

Canonical flow (copy)

export RPC_URL="https://sepolia.base.org"
export CONTRACT="0x..."
export PRIVATE_KEY="0x..."

PRICE_WEI=$(cast call --rpc-url "$RPC_URL" "$CONTRACT" "getPrice(string,uint8)(uint256)" "yourname" 1 | awk '{print $1}')
cast send --rpc-url "$RPC_URL" --private-key "$PRIVATE_KEY" "$CONTRACT" "register(string,uint8)" "yourname" 1 --value "$PRICE_WEI"
cast send --rpc-url "$RPC_URL" --private-key "$PRIVATE_KEY" "$CONTRACT" "setCID(string,string)" "yourname" "bafy..."

curl -i https://yourname.ipns.io/
