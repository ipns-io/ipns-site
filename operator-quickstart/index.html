Operator Quickstart (1 Page)

Run an independent gateway that resolves the same onchain registry and serves names on your own domain.

What You Get

  • yourname.yourdomain.tld resolution from the same IPNS registry contract
  • Wildcard host routing + path mode fallback
  • Optional apex landing page via LANDING_CID

1) Deploy Worker

Deploy the reference worker:

wrangler deploy

Set Worker environment variables:

APEX_DOMAIN=yourdomain.tld
BASE_RPC_URL=https://mainnet.base.org
CONTRACT_ADDRESS=0x...your_registry_contract
IPFS_GATEWAY_ORIGIN=https://ipfs.io/ipfs
ENABLE_DNSLINK=false

2) Add Routes + DNS

Cloudflare Worker Routes:

yourdomain.tld/*
*.yourdomain.tld/*

Cloudflare DNS (proxied / orange cloud):

A  @  192.0.2.1
A  *  192.0.2.1

3) Verify

curl -sI https://yourdomain.tld/ | grep -iE "x-ipfs-path|cache-control|http/"
curl -sI https://yourname.yourdomain.tld/ | grep -iE "x-ipfs-path|http/"

Expected: HTTP 200 for registered names, 404 for unregistered names.

4) Optional Local Run (Node)

BASE_RPC_URL="https://mainnet.base.org" \
CONTRACT_ADDRESS="0x...your_registry_contract" \
APEX_DOMAIN="yourdomain.tld" \
IPFS_GATEWAY_ORIGIN="https://ipfs.io/ipfs" \
node server.js
curl http://localhost:8787/yourname/

Operator Policy

  1. Do not rewrite registry results.
  2. Publish your gateway domain and resolver config.
  3. Document your uptime/SLA expectations.
  4. Keep cache TTL short enough to reflect CID updates quickly.
